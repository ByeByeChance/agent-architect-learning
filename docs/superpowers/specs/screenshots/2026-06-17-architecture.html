整体架构：你负责的部分 vs 现成的部分

离职状态下的最小可行 Agent 系统

Agent 体验架构师 — 系统分层
🎨 前端层 ← 你主攻
Agent 交互界面 · HITL 审批 · 推理过程可视化 · 工作流状态面板
↕ API / WebSocket / SSE
🔌 MCP 工具层 ← 你构建
Node.js/TS MCP Servers · 自定义工具 · 数据资源 · Prompt 模板
↕ MCP 协议
🧠 Agent 推理层 ← 现成的
Claude API / OpenAI API · 工具调用 · 上下文管理 · 推理链
↕ LLM 推理
☁️ 模型层 ← 不用管
Claude / GPT 等大语言模型

你负责 上面两层（前端 + MCP），下面两层直接用 API。这样不依赖任何团队后端。

关键设计决策

为什么 MCP 而不是自己搭 Agent 后端？

MCP 是标准协议，Codex、Claude Desktop 等客户端已经实现了工具调用和上下文管理。你只需要写 Server 暴露工具，不用管 agent 怎么调度、上下文怎么管理。这部分复杂度先外包给现成客户端。

为什么 Node.js/TS 而不是 Python？

MCP 官方 TypeScript SDK 很成熟，你的 Node.js 背景直接用。前端和 Server 用同一语言，认知负担最小。

API 成本？

Claude API 和 OpenAI API 按 token 计费。开发测试阶段用量很小，一个月几十块够用。后期做产品才需要考虑成本治理（那是 Autonomous Optimization Architect 的事了）。
