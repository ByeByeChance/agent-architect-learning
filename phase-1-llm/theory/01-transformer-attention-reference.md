# Transformer 与 Attention 机制 — 标准参考

> 此文件为理论参考，你的个人笔记保留在 `01-transformer-attention.md` 中。

## 1. 为什么要有 Attention？

**RNN/LSTM 的两大缺陷：**

- **长距离依赖衰减**：RNN 逐个 token 顺序处理，每步依赖上一步的隐藏状态。当序列很长时（如 100 个 token），开头的信息经过 100 步传递后已严重衰减，模型实际上"忘记"了远距离的上下文。
- **无法并行训练**：因为每一步都依赖前一步，GPU 只能串行执行。这就是为什么 RNN 时代的模型（如 LSTM 翻译模型）训练极慢。

**Attention 怎么解决：**

Attention 让每个输出位置直接"看到"输入序列全部位置，并计算出每个位置与当前位置的**相关程度**（注意力权重）。信息传递路径长度从 O(n) 降到了 O(1)——不再需要经过中间步骤层层传递。同时所有位置的计算互相独立，可以完全并行。

> 类比：RNN 是逐字读一本书，读到第 100 页时已经忘了第 1 页；Attention 是把整本书摊开在桌上，同时看到所有页面，然后决定"我现在需要看哪里"。

## 2. Self-Attention 怎么算？

Self-Attention 的核心公式：

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

**Q、K、V 是什么？**

- **Q (Query，查询)**：来自"当前这个词"——"我在找什么？"
- **K (Key，键)**：来自"所有词"——"我身上有什么特征？"  
- **V (Value，值)**：也来自"所有词"——"如果有人关注我，我传递什么信息？"

三者通过可学习的权重矩阵从同一个输入向量投影得到：
```
Q = X × W_Q
K = X × W_K  
V = X × W_V
```

**计算步骤：**

1. **Score**：Q 与每个 K 做点积，得到当前词与每个词的相关性分数。分数越高 = 越相关。
2. **Scale**：除以 $\sqrt{d_k}$（关键维度的平方根），防止点积过大导致 softmax 梯度消失。
3. **Softmax**：将分数转为概率分布（总和为 1），每个值表示"应该花多少注意力在这个词上"。
4. **Weighted Sum**：用 softmax 得到的权重对 V 加权求和，得到最终输出。

> 类比：你在图书馆找"机器学习"相关的书（Q），每本书有标签（K），你要的书有具体内容（V）。你扫描所有标签，决定哪几本最相关，然后从它们的内容中综合出答案。

## 3. Multi-Head Attention 解决了什么？

单头 Attention 只学到**一种**关注模式。但语言是丰富的——一个词与另一个词的关系可以是：语法关系、语义关系、指代关系、位置关系等等。

**Multi-Head Attention = 多组并行的 Self-Attention**

每组（一个 head）有自己的 W_Q、W_K、W_V 矩阵，可以学到不同类型的注意力模式：

- Head 1 可能关注"主语-谓语"的语法关系
- Head 2 可能关注代词的前指关系（"它"指代什么）
- Head 3 可能关注语义相似性

所有 head 独立计算，最后将结果拼接（concat）并线性投影：

$$\text{MultiHead} = \text{Concat}(\text{head}_1, ..., \text{head}_h) \times W_O$$

论文中使用 8 个 head，每个 head 的维度是 d_model / 8 = 64。

## 4. Transformer 的整体结构

### Encoder（编码器）
- 输入：源序列（如英文句子）
- 自注意力层（Self-Attention）：让每个词看到整个输入序列
- 前馈网络（Feed-Forward）：对每个位置独立做非线性变换
- 输出：包含上下文信息的向量序列

### Decoder（解码器）
- 输入：已生成的部分输出 + Encoder 的输出
- **Masked Self-Attention**：只能看到当前位置**之前**的词（防止"作弊"看到未来）
- **Cross-Attention**：Q 来自 Decoder，K/V 来自 Encoder——让 Decoder"参考"输入序列
- 前馈网络
- 输出：下一个词的概率分布

### Positional Encoding（位置编码）
Transformer 没有 RNN 的时序结构，它把所有词"平铺"同时看，**不知道哪个词在前、哪个在后**。位置编码就是给每个位置注入一个独特的向量，让模型知道词的顺序。

论文使用的是正弦/余弦函数：
$$PE(pos, 2i) = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$
$$PE(pos, 2i+1) = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$

> 现代模型（GPT、LLaMA）更多使用**可学习的位置编码 (RoPE)**，效果更好。

## 5. 为什么 Transformer 能规模化？

**并行训练**
RNN 必须串行，Transformer 的 Self-Attention 可以一次性对所有位置并行计算。这使得在数千块 GPU 上训练万亿级别参数的模型成为可能。

**梯度稳定性**
残差连接（Residual Connection）+ Layer Normalization 让梯度在深层网络中不会消失或爆炸。这也是为什么可以堆叠几十上百层。

**Scaling Law**
研究发现，**模型规模、数据量、计算量**与模型性能之间存在可预测的幂律关系。简单说：只要把模型做大、数据给够、算力管饱，性能就会持续提升——"大力出奇迹"的数学支撑。

## 6. 怎样向外行人解释 ChatGPT 为什么"懂"你？

ChatGPT 的工作原理可以这样理解：它读遍了互联网上几乎所有公开的文本（相当于"读完了人类有史以来写过的所有书和网页"），在这个过程中学会了**词与词之间的关联模式**——"太阳"后面大概率跟"升起来"，"我饿了"后面大概率跟"去吃饭"。

当它"读"你的问题时，它不是真正"理解"了你在想什么，而是在它巨大的知识库中，**找到和你的问题最匹配的回答模式**，然后一个词一个词地拼接出一个合理回答。它没有意识、没有感受、不会"思考"——它只是在做极其精密的模式匹配。但这个模式匹配能力强大到可以通过大学考试。

## 自测参考答案

1. **Q、K、V 从哪里来？**  
   都来自输入向量 X，通过三个不同的可学习权重矩阵 W_Q、W_K、W_V 做线性投影得到。

2. **为什么 Attention 比 RNN 好？**  
   Attention 让信息传递路径为 O(1)（直接看到所有位置），RNN 为 O(n)（逐层传递）；Attention 可完全并行训练，RNN 必须串行。

3. **Multi-Head 的 Multi 是什么意思？**  
   多个并行的注意力"头"，每个头学习不同类型的语义/语法关系，最后拼接合并——类似多个专家同时审阅同一份材料，各从不同角度给意见。

4. **Positional Encoding 解决什么？**  
   Transformer 没有序列感——它把所有词同时处理，无法区分前后顺序。位置编码给每个位置嵌入独特的向量，让模型知道词的顺序。

5. **Decoder-only vs Encoder-Decoder 区别？**  
   Encoder-Decoder（如 BERT+T5）有独立编码器和解码器，输入先被编码再被解码成输出，适合翻译等需要理解两种语言的任务。Decoder-only（如 GPT）只有解码器，通过预测下一个 token 来生成文本，训练更简单高效，是目前主流大模型架构。
